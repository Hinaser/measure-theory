# AGENTS.md — 厳密な確率論のための測度論入門

このリポジトリは、Docusaurus 上で公開する日本語の数学教材『厳密な確率論のための測度論入門』を、複数の GPT-5.5 サブエージェントで並行執筆するための作業指示書です。

## 目的

読者が「確率論を厳密に理解するために必要な測度論」を、集合・σ-代数・測度・可測関数・ルベーグ積分から始めて、確率空間・確率変数・期待値・独立性・収束・条件付き期待値まで一貫して学べる教材を作る。

本教材は測度論単体の網羅的教科書ではなく、確率論を厳密化するための測度論に重点を置く。

## 想定読者

- 大学初年級の微積分・線形代数・集合記法には触れたことがある。
- 素朴な確率論、離散確率、連続分布、期待値、分散は知っている。
- 「確率変数は可測関数」「期待値はルベーグ積分」という説明で一度つまずいた人。
- 暗号理論、統計学、機械学習、情報理論、確率過程などの理論的土台を固めたい人。

## 全体のトーン

- 日本語で書く。
- 厳密だが、最初の説明は直感から入る。
- 定義・命題・定理・証明・例・反例・演習を明確に分ける。
- 「なぜこの概念が確率論で必要なのか」を各章で必ず説明する。
- 数学的な嘘や過度な簡略化は避ける。ただし初出時には「まずはこの直感でよい」と明示してよい。
- 読者が混乱しやすい点は `:::warning[注意]` や `:::tip[直感]` の admonition で補足する。

## 数式と記法

LaTeX 数式を使う。サイト側で `remark-math` と `rehype-katex` または同等の数式レンダリングが有効になっている想定で、本文では次を使う。

インライン数式:

```md
確率空間は $(\Omega, \mathcal{F}, P)$ と書く。
```

ブロック数式:

```md
$$
\mathbb{E}[X] = \int_\Omega X \, dP
$$
```

MDX と衝突しやすい文字に注意する。

- 不等号 `<` `>` は文中で必要ならコード化するか数式内に置く。
- JSX に見える `{}` を本文でむやみに使わない。
- 集合の内包表記は数式ブロックに置くのが安全。

## 定義・定理・例の書式

統一感を出すため、次の書式を使う。

```md
:::info[定義：σ-代数]
集合 $\Omega$ の部分集合族 $\mathcal{F} \subset 2^\Omega$ が次を満たすとき、$\mathcal{F}$ を $\Omega$ 上の σ-代数という。

1. $\Omega \in \mathcal{F}$
2. $A \in \mathcal{F}$ ならば $A^c \in \mathcal{F}$
3. $A_1, A_2, \ldots \in \mathcal{F}$ ならば $\bigcup_{n=1}^{\infty} A_n \in \mathcal{F}$
:::
```

```md
:::tip[直感]
σ-代数は「確率を割り当てることを許された事象の集まり」である。
:::
```

```md
:::warning[注意]
すべての部分集合に確率を与えられるとは限らない。
:::
```

## 演習問題の方針

各章に 5〜12 問を置く。

難易度ラベルを付ける。

- ★ 基本確認
- ★★ 標準
- ★★★ 発展

例:

```md
1. ★ $\mathcal{F}$ が σ-代数ならば $\emptyset \in \mathcal{F}$ であることを示せ。
2. ★★ 可算個の σ-代数の共通部分が σ-代数であることを示せ。
3. ★★★ 生成される σ-代数の最小性を示せ。
```

解答は本文直下に置かず、必要なら同じ章の末尾に `## 演習問題のヒント` を置く。完全解答は付録Eの担当に渡す。

## 並行執筆時の依存関係

各エージェントは、担当範囲外の内容を過度に再定義しない。

- 第1〜2章は記法の基準を作る。
- 第3〜5章は σ-代数・測度・ルベーグ測度の基準定義を作る。
- 第6〜8章は可測関数・積分・期待値の基準定義を作る。
- 第10章以降は、それ以前の定義に依存して確率論を展開する。
- 発展章は、未導入の定理を使う場合、必ず「ここでは結果だけ使う」と明示する。

## 内容の重複を避ける

同じ概念を複数章で扱う場合、初出章では定義と証明を詳しく書き、後続章では短く参照する。

例:

- σ-代数の定義は第3章で詳述する。
- 確率空間の章では「第3章のσ-代数を事象の集まりとして使う」と説明する。
- 期待値の章では、ルベーグ積分の構成そのものを再証明しない。

## 厳密性の基準

- 定義域・値域を省略しない。
- 「ほとんど至るところ」「ほとんど確実に」は必ず測度または確率測度を明示する。
- 独立性では、事象・σ-代数・確率変数の独立性を混同しない。
- 分布と密度を混同しない。密度は常に存在するとは限らない。
- 条件付き期待値は関数ではなく同値類として扱う場面があることに注意する。
- 「同じ確率変数」と「同分布」は明確に区別する。

## 禁止事項

- 「可測性は細かい話なので無視してよい」と断言しない。
- 連続分布では確率変数が必ず密度を持つかのように書かない。
- すべての集合にルベーグ測度が定義できるかのように書かない。
- Radon-Nikodym 定理や Kolmogorov 拡張定理を、条件なしに使える万能定理として書かない。
- 証明していない強い定理を、読者が当然知っている前提で使わない。
